OVERVIEW

Each character sprite sheet in the Wonderdot series has the same layout.

  1. Row 1 : Idle
  2. Row 2 : Select/action
  3. Row 3-5 : Moving
  4. Row 6-8 : Attacking
  5. Row 9-11 : Hit
  6. Row 9-11, last frame : Still frames

The sprite sheets are formatted in 48*48 pixel cells, so that it’s easy to convert the sheet into whatever format you need.

The character is always positioned as if it was occupying the centre 16*16 cell.

ROW 1 - Idle

 

-Idle frames in the WonderDot style always respect the 16x16 grid, so that all units are easily visible while stationary. Exceptions are made for large characters, which are up to 24 pixels tall.

-Ping-pong style animation. 1 - 2 - 3 - 2 (and repeat)

-Frames 1 and 3 require a longer delay than frame 2.
You may prefer a longer or shorter pause at frame 1 & 3, depending on the style you’re after.

ROW 2 - Selected / Action

Use same configuration as idle animation.

ROW 3, 4, 5 - Movement (down, side, up)

 

-Simple loop animation. Frames have a delay of 100ms in this example. You may want to tweak this number according to the character movement speed.

-Flip side frames for left facing animations.

Notes:

  • Some character animations have offsets embedded into the animations, or already extend quite far. In these cases ignore the suggestion C.
  • Depending on the type of game you’re making, you’ll likely want to adjust these values.
    For example, in an action game you may want the player’s actions to be more responsive (take out the delay in frame 1), and the enemy’s actions easier to react to (more frame 1 delay).

Archer Units

To better make use of the four attack frames, archer units have been designed to have longer delay on the 2nd frame, rather than the first.

ROW 9,10,11 - Hit

Frame order: 1,2,3,1

-Frame 1 is build-up frame. Give it a little extra delay to build anticipation for the hit.

-Frame 2, 3 are the hit frames.

-Ease back with frame 1 again.

Frame 4 is an all-purpose idle frame. Use these as you like.

You can loop frames 2,3 indefinitely to extend the animation. Could be useful if being hit by a long spell animation (etc.).
